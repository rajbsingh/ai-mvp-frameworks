# MVP Scope Template

This template helps founders and product teams define a focused MVP for an AI SaaS product.

The goal of an MVP is not to build everything. The goal is to validate whether the product solves a real problem for a real user.

---

## 1. Product Idea

Write the product idea in one sentence.

Example:

An AI assistant that helps small businesses review contracts and identify key risks before legal review.

---

## 2. Target User

Define the first user group clearly.

Questions:

* Who has the problem?
* What role do they have?
* How often do they face this problem?
* Are they already paying for a solution?
* Can they make or influence a buying decision?

---

## 3. Core Problem

Describe the painful problem.

A strong MVP problem should be:

* Frequent
* Expensive
* Time-consuming
* Risky
* Important enough to solve
* Clear enough to explain

---

## 4. Desired Outcome

Define what success looks like for the user.

Examples:

* Save time
* Reduce risk
* Understand documents faster
* Automate repetitive work
* Improve decision-making
* Prepare better first drafts

---

## 5. Must-Have MVP Features

List only features required to test the core value.

Example:

* Upload document
* Generate summary
* Extract key points
* Flag risks
* Produce structured output
* Allow human review

---

## 6. Nice-to-Have Later

List features that can wait.

Examples:

* Team dashboard
* Advanced analytics
* Integrations
* Workflow history
* Admin controls
* Multi-language support
* Payment automation

---

## 7. Avoid for Now

List features that should not be built in the first version.

Examples:

* Full enterprise platform
* Complex automation
* Too many integrations
* Too many user roles
* Too many industries
* Fully autonomous decisions
* Complex AI agents before validation

---

## 8. Validation Plan

Before building too much, test demand.

Validation methods:

* Landing page
* User interviews
* Demo walkthrough
* Clickable prototype
* Manual concierge MVP
* Paid pilot
* Waitlist
* LinkedIn outreach

---

## 9. Success Metrics

Define how MVP success will be measured.

Examples:

* Number of interested users
* Number of demo requests
* Number of pilot users
* Repeat usage
* Time saved
* Willingness to pay
* User feedback quality

---

## 10. MVP Decision

After validation, decide:

* Continue
* Improve
* Narrow the target user
* Change the workflow
* Pause the idea
* Build the next version

---

## Conclusion

A strong MVP is focused, testable, and useful.

The best AI SaaS MVPs solve one painful problem well before expanding into a larger platform.
