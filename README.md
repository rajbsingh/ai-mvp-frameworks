## Purpose

Many AI startup ideas fail because founders build too much before validating demand.

This repository shares practical frameworks to move from idea to MVP to validation.

---

## MVP Planning Framework

### 1. Define the User

Who has the problem?

### 2. Define the Pain

What painful, frequent, or expensive problem are they facing?

### 3. Define the Outcome

What result should the AI product help them achieve?

### 4. Define the Smallest Useful Product

What is the smallest version that can prove value?

### 5. Define the Risk

What can go wrong technically, legally, commercially, or operationally?

### 6. Validate Before Scaling

Test with real users before building a full platform.
